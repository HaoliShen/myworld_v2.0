# **代码构建指令 \- Update V1: 世界初始化与调试回退**

**目标:** 完善游戏启动流程。实现 WorldManager 的初始化逻辑，使其能够连接数据库、生成玩家并启动流式加载。同时在 SaveSystem 中增加调试用的“默认世界”生成逻辑，以便直接运行场景时不会报错。

## **1\. 修改目标: Scripts/Core/SaveSystem.gd**

**说明:** 我们需要一个变量来存储当前要加载的世界名，以及一个辅助函数来处理“没有存档时直接运行”的情况。

**请添加/修改以下代码:**

1. **添加属性:**  
   \# 当前加载的世界名称 (默认为空)  
   var current\_world\_name: String \= ""

2. **添加函数 load\_or\_create\_debug\_world():**  
   * **功能:** 检查是否存在名为 "DebugWorld" 的存档。如果存在则返回该名字；不存在则调用 create\_new\_world 创建它。最后将 current\_world\_name 设置为 "DebugWorld"。  
   * **目的:** 供 WorldManager 在 current\_world\_name 为空时调用。

## **2\. 修改目标: Scripts/Systems/WorldManager.gd**

**说明:** 实现 \_ready() 函数，作为整个游戏关卡的启动入口。

**请实现 \_ready() 函数，逻辑如下:**

1. **存档上下文检查 (Debug Fallback):**  
   * 检查 SaveSystem.current\_world\_name 是否为空字符串。  
   * 如果为空（说明是直接运行场景，非主菜单进入），调用 SaveSystem.load\_or\_create\_debug\_world() 获取默认世界名。  
2. **数据库初始化:**  
   * 调用 SaveSystem.get\_world\_db\_path(...) 获取数据库路径。  
   * 调用 RegionDatabase.init\_database(...) 初始化连接。  
3. **玩家生成 (Spawn Player):**  
   * (此处简化逻辑，暂不读取数据库中的玩家旧位置，直接使用默认坐标)  
   * 定义默认出生点 var spawn\_pos \= Vector2(0, 0)。  
   * 加载 res://Scenes/Entities/Player.tscn 资源。  
   * 实例化 Player，设置 global\_position \= spawn\_pos。  
   * 获取场景中的 %EntityContainer 节点（请确保在 World.tscn 中设置了该节点的 Unique Name，或者使用路径查找），将 Player 添加为其子节点。  
4. **启动流式加载 (Kickoff):**  
   * 计算玩家所在的区块坐标：var start\_chunk \= MapUtils.world\_to\_chunk(spawn\_pos)。  
   * 调用 update\_chunks(start\_chunk)，强制刷新一次周围环境。  
5. **信号连接:**  
   * 监听 SignalBus.player\_entered\_new\_chunk，绑定到 update\_chunks 函数，确保后续移动能自动加载地图。

## **3\. 修改目标: Scenes/World/World.tscn (编辑器操作建议)**

**说明:** 为了让代码能找到节点，需要对场景树进行微调。

**操作提示:**

1. 选中 World/Environment/EntityContainer 节点。  
2. 在编辑器中将其勾选为 **"Access as Unique Name"** (即节点名旁边会出现 % 符号)，方便脚本中通过 %EntityContainer 访问。
