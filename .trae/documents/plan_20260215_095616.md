# 地形系统升级方案：多层级地形与分步生成

本方案旨在将现有的单层地形系统升级为"基础地形 + 4个高度层级"的 5 层架构，以支持更丰富的地形表现（如草地上的山脉、沙地上的丘陵）。

## 1. 常量配置 (`Constants.gd`)

需要新增层级定义、地形 ID 常量以及图块资源配置。

### 1.1 渲染层级枚举更新

扩展 `Layer` 枚举，明确 5 个地形层级。

```gdscript
enum Layer {
    # --- 地形层 (Z-Index: -100 ~ -96) ---
    TERRAIN_BASE = 0,  # 基础层 (泥土/草地/沙地)
    TERRAIN_EXH1 = 1,  # 高度层 1
    TERRAIN_EXH2 = 2,  # 高度层 2
    TERRAIN_EXH3 = 3,  # 高度层 3
    TERRAIN_EXH4 = 4,  # 高度层 4

    # --- 物体层 (Z-Index: 0, Y-Sort Enabled) ---
    DECORATION = 5,    # 装饰 (花草)
    OBSTACLE = 6       # 障碍 (树石)
}
```

### 1.2 地形 ID 定义

将地形 ID 分为"基础类"和"高度类"。

```gdscript
# 基础地形 ID
const T_BASE_DIRT: int = 0
const T_BASE_GRASS: int = 1
const T_BASE_SAND: int = 2

# 高度地形 ID (统一使用一种山脉材质，或区分不同风格)
const T_EXH_1: int = 10
const T_EXH_2: int = 11
const T_EXH_3: int = 12
const T_EXH_4: int = 13
```

### 1.3 资源配置表

新增 `TERRAIN_CONFIG` 用于集中管理 Atlas 坐标和源 ID。

```gdscript
# 地形配置表
# Key: Terrain ID -> Value: { source_id, atlas_coord(base), ... }
const TERRAIN_CONFIG: Dictionary = {
    T_BASE_DIRT:  { "source_id": 0, "atlas_base": Vector2i(0, 0) }, # 示例坐标
    T_BASE_GRASS: { "source_id": 0, "atlas_base": Vector2i(4, 0) },
    T_BASE_SAND:  { "source_id": 0, "atlas_base": Vector2i(8, 0) },
    
    T_EXH_1:      { "source_id": 0, "atlas_base": Vector2i(0, 4) },
    T_EXH_2:      { "source_id": 0, "atlas_base": Vector2i(4, 4) },
    # ...
}
```

## 2. 数据结构升级 (`ChunkData.gd`)

将原有的单一 `terrain_map` 拆解为明确的"基础地形"与"高度数据"。

### 2.1 核心属性变更

* **`base_terrain_map`** **(PackedInt32Array)**: 存储基础地形 ID (Dirt/Grass/Sand)。替代原有的 `terrain_map`。

* **`elevation_map`** **(PackedByteArray)**: 保持不变，存储高度值 (0-4)。

  * 0: 仅显示 Base

  * 1: 显示 Base + Exh1

  * 2: 显示 Base + Exh1 + Exh2

  * 以此类推。

### 2.2 接口变更

* `get/set_base_terrain(x, y, id)`: 操作基础地形。

* `get/set_elevation(x, y, level)`: 操作高度 (0-4)。

***

## 3. 生成逻辑重构 (`MapGenerator.gd`)

生成过程分为两个独立步骤，互不干扰。

### 3.1 步骤一：基础地形生成 (Base Generation)

* 使用 **Biome Noise** (温度/湿度噪声)。

* 根据噪声值决定基础地形类型：

  * 低值 -> 沙地

  * 中值 -> 泥土

  * 高值 -> 草地

* 填充 `base_terrain_map`。

### 3.2 步骤二：高度生成 (Elevation Generation)

* 使用 **Elevation Noise** (地形起伏噪声)。

* 根据噪声值决定高度层级 (0-4)。

* 填充 `elevation_map`。

* *注：此时高度不再强制绑定特定的地形 ID，而是叠加在任何基础地形之上。*

***

## 4. 渲染系统更新 (`ChunkVisual.gd` & `GlobalMapController.gd`)

渲染层需要同时处理 5 个 TileMapLayer 的数据。

### 4.1 场景结构 (`ChunkVisual.tscn`)

更新节点结构，包含 5 个地形层节点：

* `BaseLayer` (Z: -100)

* `Exh1Layer` (Z: -99)

* `Exh2Layer` (Z: -98)

* `Exh3Layer` (Z: -97)

* `Exh4Layer` (Z: -96)

* `DecorationLayer` ...

### 4.2 视觉数据计算 (`GlobalMapController.gd`)

在后台线程计算时，生成 5 份 Layer Data：

1. **Base Layer**: 根据 `base_terrain_map` 计算 Auto-tile (处理不同基础地形间的过渡，如果需要)。
2. **Exh Layers (1-4)**:

   * 遍历每个高度层 N (1\~4)。

   * 如果 `elevation_map[x,y] >= N`，则该层有图块。

   * 根据周围格子的存在情况计算 Auto-tile (处理山脉边缘)。

### 4.3 应用数据 (`ChunkVisual.gd`)

`apply_visual_data` 函数需更新，接收并应用所有 5 个地形层的数据。

## 5. 实施步骤

1. **配置更新**: 修改 `Constants.gd`，添加新常量。
2. **数据迁移**: 修改 `ChunkData.gd` 结构 (注意：这会破坏旧存档兼容性，建议重置存档)。
3. **生成器重写**: 修改 `MapGenerator.gd` 实现两步生成。
4. **渲染更新**: 修改 `ChunkVisual` 脚本及 `GlobalMapController` 的计算逻辑。
5. **验证**: 运行游戏，检查是否能看到不同基础地形上生成的山脉层叠效果。
