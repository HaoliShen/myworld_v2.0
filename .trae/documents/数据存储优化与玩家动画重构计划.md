# 实施计划

## 1. 数据存储模块 (Data Storage) 分析与优化
**现状分析**:
- **实现情况**: `RegionDatabase.gd` 实现了基于 SQLite 的区域分片存储 (`r.x.y.rg`)，符合设计文档中"分区域聚合存储"的要求。
- **存在问题**:
    1. **主线程阻塞 (Critical)**: `WorldManager.gd` 中的 `_request_chunk_data` 直接调用 `RegionDatabase.load_chunk`，这是同步 I/O 操作。当玩家移动触发新区块加载时，会导致游戏主线程卡顿 (Stuttering)。
    2. **写入效率**: `force_save_all` 循环调用 `save_chunk`，每次写入可能触发一次独立的 I/O 事务，效率较低。

**修改方案**:
- **引入多线程加载**: 修改 `WorldManager.gd`，使用 `WorkerThreadPool` 将 `RegionDatabase.load_chunk` 放入后台线程执行。
- **回调机制**: 保持现有的 `call_deferred("_on_chunk_data_ready", ...)` 机制，确保线程安全地返回主线程。

## 2. Better Terrain 使用情况检查
**检查结果**:
- **已安装**: 插件存在于 `addons/better-terrain` 目录下。
- **未使用**: 在 `GlobalMapController.gd` 中，涉及 `BetterTerrain` 的 API 调用（如 `update_terrain_cells`）目前处于**注释状态**。
- **结论**: 当前项目仅使用了 Godot 原生 `TileMapLayer` API，未启用 Better Terrain 的自动地形连接功能。

**执行步骤**:
